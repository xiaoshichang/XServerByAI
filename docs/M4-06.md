# M4-06 Client -> Gate -> Game RPC 请求链路

## 当前状态
`M4-06` 当前已经落地首条基于 `AvatarEntity` 的端到端请求链路，但在开发计划中仍保持为“开发中”，后续还会继续围绕更完整的响应链路、属性同步和更丰富的 RPC 场景迭代。

## 本条目落地范围
1. `client` 与 `server` 两侧的 `AvatarEntity` 都增加业务字段 `Weapon`。
2. sim client 新增 `set-weapon <weapon>` / `setweapon <weapon>` 命令。
3. client 通过 `ClientEntity.CallServerRpc("SetWeapon", weapon)` 发送 `ClientToServerEntityRpcMsgId = 6302`。
4. `Gate` 识别客户端 `6302`，校验当前会话已经绑定 `avatarId` 与 `gameNodeId`，再把原始 RPC 负载重封装为 `Relay.ForwardProxyCall (2005)` 转发给目标 `Game`。
5. `Game` 侧 `AvatarEntity.SetWeapon(string weapon)` 更新 `Weapon`，并输出包含 `entityId / accountId / weapon` 的日志。
6. `Game` 成功处理后，通过 `CallClientRpc("OnSetWeaponResult", weapon, true)` 回推 `6303`；client 收到成功结果后更新本地 `AvatarEntity.Weapon`。

## 关键实现

### 1. Client 侧
- `XServer.Client.Framework/Rpc/ClientEntityRpcPacketSender.cs`
  - 负责把 `ClientEntityRpcRequest` 包装为真实客户端包头。
  - 发送时自动使用当前 `ClientRuntimeState` 的包序号，并记录发包统计。
- `XServer.Client.App/App/ClientConsoleApp.cs`
  - `connect` 成功后为 runtime 配置真实的 `IClientEntityRpcSender`。
  - `disconnect` / transport 释放时清空 sender，避免离线状态继续发 RPC。
  - 新增 `set-weapon` 命令入口，支持脚本模式与交互模式。
- `XServer.Client.GameLogic/ClientGameLogicService.cs`
  - 新增 `SendSetWeaponRpc(...)`，统一完成 avatar-ready 校验与 RPC 发送。

### 2. Gate 侧
- `src/native/node/GateNode.cpp`
  - 新增 `kClientEntityRpcMsgId = 6302U`。
  - 新增 `HandleClientEntityRpcPacket(...)`。
  - 校验项包括：
    - 包头 `flags == 0`
    - `seq != 0`
    - 已存在认证后的 `ClientSessionRecord`
    - 会话未关闭
    - 已绑定 `avatarId`
    - 已绑定目标 `gameNodeId`
    - `avatarId -> sessionId` 映射仍然指向当前会话
    - 目标 `Game` inner session 已注册、已连通、可路由
  - 通过现有 `Relay.ForwardProxyCall` 编码，把客户端 RPC 转到 `Game`。

### 3. Game / Managed 侧
- `src/managed/Framework/Entities/AvatarEntity.cs`
  - `EquippedWeaponId` 已统一改名为 `Weapon`。
  - `[ServerRPC] SetWeapon(string weapon)` 成功后会：
    - 更新服务端业务状态 `Weapon`
    - 输出 server 日志
    - 调用 `CallClientRpc("OnSetWeaponResult", Weapon, true)`
- `client/XServer.Client.Framework/Entities/AvatarEntity.cs`
  - `[ClientRPC] OnSetWeaponResult(string weapon, bool succ)` 在成功时同步本地 `Weapon`。

## 使用方式

### Sim Client
```csharp
public void OnBtnClick()
{
    self.CallServerRpc("SetWeapon", "gun");
}

[ClientRPC]
public void OnSetWeaponResult(string weapon, bool succ)
{
    if (succ)
    {
        // 本地 Weapon 已更新为 weapon
    }
}
```

### Server AvatarEntity
```csharp
[ServerRPC]
public void SetWeapon(string weapon)
{
    Weapon = weapon;
    this.CallClientRpc("OnSetWeaponResult", Weapon, true);
}
```

### Console 命令
```powershell
set-weapon gun
```
