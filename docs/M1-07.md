# M1-07 设计说明（msgId 分段与命名规范）

1. 将 `msgId` 分段、命名与登记规则沉淀为 `docs/MSG_ID.md` 的长期维护文档，避免后续条目各自占号。
2. 基于当前服务器组拓扑与消息职责划分，将号段拆分为协议保留、控制面、Gate↔Game 中转、客户端接入、业务实体、运维诊断与测试预留几个大类。
3. 明确响应与错误语义优先通过 `PacketHeader.flags` 表达：响应默认复用请求 `msgId`，错误不单独创建成对 `msgId`。
4. 规定消息的规范英文名格式、跨文档登记字段与代码常量命名映射，确保 C++ 与 C# 后续实现使用同一语义。
5. 为注册/心跳、Gate↔Game 封装、会话路由、实体分发与 Stub 服务预留稳定子号段，降低后续里程碑互相冲突的概率。
6. 在 `docs/PROJECT.md` 与 `docs/PACKET_HEADER.md` 中统一引用该规范，形成单一事实来源。
