# M2-07 设计说明（ZeroMQ over TCP 主动连接端）

1. 在 `src/native/ipc` 内新增 `ZmqContext` 与 `ZmqActiveConnector`，当前阶段只覆盖“节点间主动连接端”语义；socket 类型固定为 `ZMQ_DEALER`，与后续 `M2-08` 的监听端 `ROUTER` 形成标准的 `DEALER -> ROUTER` 组合，不提前扩展为通用多模式传输层。
2. 当前项目运行时仍然是“单 owner 线程 + standalone `asio` 事件循环”的模型，因此项目侧不把 `ZMQ_FD` 直接桥接成 Asio descriptor；ZeroMQ 继续自行管理内部 I/O 线程、TCP 握手与自动重连，Asio 只负责生命周期串行化、定时调度与在主线程上驱动轮询 pump。
3. `ZmqContext` 封装进程级 `zmq_ctx_t` 句柄与基础参数（`io_threads`、`max_sockets`），由调用方显式持有并在多个连接之间共享；上下文对象必须长于基于它创建的主动连接端，避免为每条连接重复创建独立 context。
4. `ZmqActiveConnector` 构造时绑定 `asio::io_context` 与 `ZmqContext`，配置项只暴露当前确定需要的字段：远端 TCP endpoint、可选 routing id、Asio 轮询间隔、`sndhwm/rcvhwm`、重连间隔与握手超时；socket 统一设置 `ZMQ_LINGER = 0` 与 `ZMQ_IMMEDIATE = 1`，确保停止时不拖尾，且在连接尚未建立时 `Send()` 明确失败而不是无界排队。
5. 连接状态不依赖阻塞式探测，而是通过 `zmq_socket_monitor(..., "inproc://...", ...)` 订阅 monitor 事件，并由同一条 Asio 定时 pump 在 owner 线程上非阻塞读取 monitor PAIR socket；项目侧对外只收敛为 `Stopped / Connecting / Connected / Disconnected` 四种状态，自动重连仍完全交给 libzmq。
6. 业务收发统一采用单帧消息 + `ZMQ_DONTWAIT`：发送接口接受 `std::span<const std::byte>`，接收端回调也只上抛单条完整字节消息，不引入阻塞等待、额外工作线程或项目侧发送队列。若收到 multipart 消息，则主动丢弃剩余 frame 并记录错误，保持当前阶段“一个内部包对应一个 ZeroMQ message”的约束。
7. 所有公开 API 都按 owner-thread 约束设计，不引入 mutex、strand 或跨线程同步语义；状态回调与消息回调总是在绑定的 Asio 线程上执行，后续 `M3-09` / `M3-11` 的注册与心跳逻辑只需要在同一线程上串行驱动即可。
8. 自动化验证继续使用最小 native 测试可执行程序，直接用原生 `ZMQ_ROUTER` 模拟对端，至少覆盖：wildcard bind 后主动连接建立、`DEALER -> ROUTER` 收发往返、monitor 状态变化、远端断开后的状态回落，以及无效配置与未启动发送这类错误路径。
