# M1-10 设计说明（Gate↔Game 消息封装格式）

1. 将 Gate↔Game 中继封装格式沉淀为 `docs/GATE_GAME_ENVELOPE.md` 的长期维护文档，作为后续转发、响应回传与推送链路的统一约束。
2. 基于既有 `msgId` 与错误码规范，为 Gate→Game 请求中继与 Game→Gate 主动推送各分配稳定消息编号，并登记基础 Relay 错误码。
3. 明确该封装建立在“同一服务器组内 Gate↔Game 全连接、Gate↔Gate 与 Game↔Game 不直接连接”的网络假设上，不引入多跳转发语义。
4. 定义共享的中继包元数据，明确 `sessionId`、`playerId`、`clientMsgId`、`clientSeq` 与 `clientFlags` 的职责边界，避免把客户端头部原样透传到内部链路。
5. 约定 `Relay.ForwardToGame` 使用请求/响应模式，内部响应复用同一 `msgId` 与 `seq`，而客户端侧响应标志由中继包中的 `clientFlags` 表达。
6. 约定 `Relay.PushToClient` 为单向消息，默认 `PacketHeader.seq = 0`，用于 Game 经由 Gate 向客户端主动下行推送。
7. 在 `docs/PROJECT.md`、`docs/MSG_ID.md` 与 `docs/ERROR_CODE.md` 中同步登记和引用，形成单一事实来源。
