# M2-06 设计说明（网络字节序与序列化基础函数）

1. 在 `src/native/net` 中新增独立的字节序与二进制序列化基础模块，面向后续 `M2-07`/`M2-09`/`M2-11`/`M2-12` 的协议编解码复用；当前条目只提供基础整数、布尔值、字符串与字节块的有界读写，不提前实现完整消息结构或 `PacketHeader` 打包逻辑。
2. 网络字节序工具统一基于标准 C++ 实现，不直接依赖平台 `htonl`/`ntohl` 或 Winsock 头文件；通过 `std::endian` + 项目内 `byteswap` 逻辑，支持 `uint8/16/32/64`、`int32` 以及后续可能复用的枚举底层整数值。
3. 序列化接口采用面向缓冲区游标的 `BinaryWriter` / `BinaryReader`：构造时绑定一段 `std::span<std::byte>` 或 `std::span<const std::byte>`，内部仅维护当前偏移与最近一次错误码，不引入异常、堆分配或隐式扩容。
4. 当前阶段明确覆盖的基础类型包括：固定宽度整数 `uint8/16/32/64`、`int32`、`bool`（按 `uint8` 编码，`0/1` 之外视为非法）、原始字节块、`uint16 length + UTF-8 bytes` 字符串，以及 `uint16`/`uint32` 长度前缀的字节块；`string[]` 等复合结构留给后续消息编解码条目在这些基础函数之上显式循环构建。
5. 所有失败路径都要求“无部分提交”：写入缓冲区剩余空间不足、长度前缀溢出或读取到截断数据时，本次操作必须返回失败并保持偏移不变，避免后续编解码逻辑在半成功状态下继续运行。
6. 字符串基础函数只负责长度前缀与字节搬运，默认将调用方提供的 `std::string_view` / 字节内容视为 UTF-8 载荷，不在本条目引入额外的 UTF-8 语义校验；更高层字段约束和协议合法性判断留给具体消息编解码条目处理。
7. 错误反馈使用轻量 `SerializationErrorCode` + `SerializationErrorMessage()`，覆盖缓冲区不足、长度溢出、布尔值非法与输出参数为空等基础失败场景，便于后续协议层将底层失败映射到更具体的协议错误码。
8. 自动化验证使用最小 native 测试可执行程序，至少覆盖：主机序到网络序的往返一致性、序列化后的精确线缆字节布局、读写 roundtrip、长度前缀边界、缓冲区不足时偏移不前进，以及非法布尔值与截断负载被拒绝。
