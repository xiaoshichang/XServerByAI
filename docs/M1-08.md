# M1-08 设计说明（错误码范围与编码规则）

1. 将错误码范围、命名与登记规则沉淀为 `docs/ERROR_CODE.md` 的长期维护文档，避免后续模块各自发明编号。
2. 约定跨 C++ 与 C# 共用的基础表示：`0` 表示成功，非 `0` 表示失败，并保持跨语言共享的稳定数值语义。
3. 基于当前工程边界划分通用、协议、网络、Inner 网络、互操作、基础设施与业务域错误号段，给后续 M2/M5 条目预留稳定空间。
4. 明确错误语义通过 `PacketHeader.flags.Error` 表达，不通过新增 `msgId` 派生错误响应；协议消息体中的 `errorCode` 仅承载具体失败原因。
5. 规定规范英文名、代码常量映射与登记模板，确保 native 与 managed 后续实现使用同一套编号与命名。
6. 在 `docs/PROJECT.md`、`docs/PACKET_HEADER.md` 与 `docs/MSG_ID.md` 中统一引用该规范，形成单一事实来源。
