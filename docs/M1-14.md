# M1-14 设计说明（分布式实体架构概念与职责）

1. 将 C# 业务层的分布式实体架构沉淀为 `docs/DISTRIBUTED_ENTITY.md` 的长期维护文档，作为后续 `M5-03` 至 `M5-16` 的统一语义基线。
2. 明确 `ServerEntity` 与 `ServerStubEntity` 的概念差异：前者承载玩家、房间等具备稳定业务身份与状态的实体，后者承载匹配、聊天、排行榜等全局服务语义。
3. 约定实体的单活 ownership 规则：同一个逻辑实体在任意时刻只允许由一个 Game 节点持有活动实例，Gate 与 GM 不直接持有业务实体状态。
4. 明确默认业务链路为 `session -> player entity -> room entity / other server entity / stub service`，并要求实体路由建立在 `docs/SESSION_ROUTING.md` 已定义的会话与 `playerId` 语义之上。
5. 定义实体的职责边界、执行模型与生命周期约束，包括串行消息处理、Tick 驱动、脏标记、持久化钩子与异步依赖边界，避免后续条目各自发明不兼容语义。
6. 在 `docs/PROJECT.md`、`docs/SESSION_ROUTING.md`、`docs/MSG_ID.md` 与 `docs/ERROR_CODE.md` 中同步补充引用，确保总览、路由、消息号段与错误码责任域都回到同一事实来源。
