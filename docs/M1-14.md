# M1-14 设计说明（分布式实体架构概念与职责）

1. 将 C# 业务层的分布式实体架构沉淀为 `docs/DISTRIBUTED_ENTITY.md` 的长期维护文档，作为后续 `M5-03` 至 `M5-16` 的统一语义基线。
2. 明确 `ServerEntity` 与 `ServerStubEntity` 的概念差异，并补充 `ServerEntity` 的迁移属性划分：`PlayerEntity` 这类实体可迁移，`SpaceEntity` 这类实体不可迁移；`ServerStubEntity` 在启动时就固定承载 Game，运行期不迁移。
3. 约定实体的单活 ownership 规则：同一个逻辑实体在任意时刻只允许由一个 Game 节点持有活动实例，Gate 与 GM 不直接持有业务实体状态。
4. 明确实体 RPC 的两种寻址方式：不可迁移实体通过静态 `Mailbox` 寻址，可迁移实体通过动态 `Proxy` 寻址，且 `Proxy` 调用需要先转发到 Gate 再解析当前所在 Game。
5. 明确默认业务链路为 `session -> PlayerEntity Proxy -> SpaceEntity Mailbox / other server entity / ServerStubEntity Mailbox`，并要求实体路由建立在 `docs/SESSION_ROUTING.md` 已定义的会话与 `playerId` 语义之上。
6. 定义实体的职责边界、执行模型与生命周期约束，包括串行消息处理、Tick 驱动、脏标记、持久化钩子与异步依赖边界，避免后续条目各自发明不兼容语义。
7. 在 `docs/PROJECT.md`、`docs/SESSION_ROUTING.md`、`docs/MSG_ID.md` 与 `docs/ERROR_CODE.md` 中同步补充引用，确保总览、路由、消息号段与错误码责任域都回到同一事实来源。
