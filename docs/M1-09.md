# M1-09 设计说明（进程注册与心跳消息结构）

1. 将 Gate/Game 向 GM 发起的注册与心跳消息结构沉淀为 `docs/PROCESS_CONTROL.md` 的长期维护文档，作为后续控制面实现的字段基线。
2. 基于已有 `msgId` 与错误码规范，为注册与心跳各分配稳定的控制面消息编号，并登记核心失败码，避免 M2/M3 阶段重复定义。
3. 明确请求、成功响应与失败响应的消息体结构，约定响应复用请求 `msgId`，失败通过 `PacketHeader.flags.Error` 与 `errorCode` 表达。
4. 定义共享结构 `Endpoint`、`LoadSnapshot` 与 `nodeId` 语义，支持后续注册表、路由下发与负载上报条目直接复用。
5. 固化当前服务器组模型：每组固定 `1 GM + N Gate + M Game`，其中 Gate/Game 使用 `NodeID`（如 `Gate0`、`Game1`）作为稳定逻辑身份；连接后先注册、注册成功后再周期心跳。
6. 固化当前默认心跳时序：默认心跳间隔 `5000ms`、超时 `15000ms`，并允许 GM 在响应中协商覆盖。
7. 在 `docs/PROJECT.md`、`docs/MSG_ID.md` 与 `docs/ERROR_CODE.md` 中同步登记和引用，形成单一事实来源。
