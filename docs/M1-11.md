# M1-11 设计说明（KCP 参数配置项与默认值）

1. 将 Gate 侧 KCP 参数项、默认值与调优边界沉淀为 `docs/KCP_CONFIG.md` 的长期维护文档，作为后续 `M4-01` / `M4-02` 接入时的默认配置基线。
2. 只定义当前确定且与 KCP 算法直接相关的参数，不把应用层心跳、会话踢出或鉴权超时混入 KCP 配置范围。
3. 采用“低延迟但保留基本拥塞控制”的默认档位：启用 `nodelay`，保持较短 `interval` 和 `fastResend`，但默认不关闭拥塞窗口。
4. 约定逻辑配置项名称、类型、默认值、可接受范围与对应 KCP 字段映射，便于后续配置加载与封装代码直接落地。
5. 明确 `mtu`、窗口、`minRto`、`deadLink` 与 `streamMode` 的默认取值，并解释这些值与场景型实时交互目标的关系；在房间类型游戏中可将 `space` 理解为房间。
6. 在 `docs/PROJECT.md` 中统一引用该规范，形成单一事实来源。
