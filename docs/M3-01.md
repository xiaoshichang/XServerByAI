# M3-01 设计说明（统一 `xserver-node` 入口与节点角色分派）

1. 保持统一入口命令行为 `xserver-node <configPath> <nodeId>`，但把参数校验与节点构造收敛到 `NodeCreateHelper`。`main()` 只负责串联 `ParseCommandLine()`、`CreateNode()`、`Init()`、`Run()`、`Uninit()`，并统一映射进程退出码。
2. `nodeId` 继续只用于选择配置实例，允许的最小集合为 `GM`、`Gate<index>`、`Game<index>`。入口层基于 `nodeId` 做静态分派，`ServerNode::Init()` 再基于选中的 `NodeConfig` 派生类型确认节点角色，避免入口层和配置层出现两套身份来源。
3. `src/native/node` 统一承载 GM、Gate、Game 三类节点实现，不再拆成独立角色入口库。`NodeCreateHelper` 只负责根据 `nodeId` 创建 `GmNode`、`GateNode` 或 `GameNode`，后续角色能力扩展全部落在各自节点子类内部。
4. 节点公共运行时状态统一收敛到 `ServerNode` 基类，而不是额外的 runtime context。配置加载结果、当前进程 `pid`、日志器、主事件循环与最近错误都由节点自己持有，使 `main()` 与具体角色实现之间只有一个稳定边界。
5. 生命周期统一依赖 `ServerNode` 的 `Init()`、`Run()`、`Uninit()`。基类负责加载 `ClusterConfig`、选择 `NodeConfig`、初始化日志与事件循环，子类通过 `OnInit()`、`OnRun()`、`OnUninit()` 扩展角色行为，保证后续 `InnerNetwork`、`ClientNetwork` 与注册心跳都能沿同一骨架演进。
6. 错误传播采用单层节点错误码：公共入口和节点实现都返回 `NodeErrorCode`，详细文本通过 `last_error_message()` 暴露给入口打印，避免额外 runtime error wrapper 再做一次阶段封装。
7. 自动化验证优先覆盖可复用的入口和生命周期边界：至少验证参数不足与非法 `nodeId` 失败、`GM`/`Gate0`/`Game0` 能构造出正确节点类型，以及 `ServerNode` 生命周期在成功与失败路径下都能保持稳定清理语义。
