# M1-12 设计说明（会话与路由数据模型）

1. 将 Gate 侧客户端会话、路由绑定与 Game 路由目录的核心数据模型沉淀为 `docs/SESSION_ROUTING.md` 的长期维护文档，作为后续会话管理、断线清理与实体路由的统一语义基线。
2. 明确 `sessionId`、`playerId`、`gameInstanceId` 与 `gameRegistrationId` 的职责边界，区分“稳定逻辑身份”与“当前有效注册租约”，避免 Game 重启后的路由歧义。
3. 定义 `SessionState`、`RouteState` 与 `GameRouteState` 等最小状态枚举，覆盖会话创建、鉴权绑定、固定路由、路由失效与关闭流程。
4. 约定 `SessionRecord`、`RouteTarget` 与 `GameDirectoryEntry` 等共享结构，以及 Gate 必备的主索引和反向索引，支持后续会话清理、按 Game 批量失效与实体路由扩展。
5. 明确当前阶段采用“会话生命周期内固定绑定”的默认策略：已绑定会话不做静默迁移，Game 重启或租约失效时进入 `RouteLost`，等待后续条目单独定义恢复策略。
6. 在 `docs/PROJECT.md`、`docs/GATE_GAME_ENVELOPE.md`、`docs/MSG_ID.md` 与 `docs/ERROR_CODE.md` 中同步引用和细化相关保留号段说明，形成单一事实来源。
