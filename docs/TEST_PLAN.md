# 测试计划（里程碑版）

本文档基于 `docs/DEVELOPMENT_PLAN.md` 的里程碑划分，按阶段给出测试目标、范围与交付物。当前不引入具体测试框架选型，仅描述测试策略与执行要求。

**目标**
1. 保证架构在开发期单机多进程、部署期多机多进程场景下可用、可测、可回归。
2. 逐步验证 C++ 基础运行时、KCP 会话与路由、C# 分布式实体框架与互操作闭环。
3. 在闭环阶段建立稳定性与性能基线，形成可追踪的质量记录。

**范围**
包含：
1. 规范与协议一致性验证。
2. C++ 运行时基础模块。
3. 进程注册与 Inner 网络。
4. KCP 会话、路由、转发链路。
5. C# 实体框架与互操作。
6. 端到端闭环、稳定性与性能基线。

不包含：
1. MMO 级别横向扩展。
2. 跨机房容灾、跨地域多活与更高阶分布式治理能力。
3. 业务玩法完整实现。

**测试策略**
1. M1 以文档评审、规范一致性与目录/依赖落点检查为主。
2. M2/M5 以模块级功能与边界验证为主。
3. M3/M4 以集成链路与异常场景验证为主。
4. M6 以端到端、稳定性、性能基线与多机部署冒烟验证为主。

**里程碑计划**

M1 工程骨架与规范
目标：确保协议、命名、目录、互操作签名以及第三方依赖落点一致且可执行。
覆盖：目录结构与第三方依赖落点、包头结构、msgId 规则、错误码范围、注册/心跳结构、KCP 参数默认值、会话与路由模型、C++↔C# 入口签名约定。
测试活动：文档评审、规范一致性检查、接口签名对齐核对、风险项记录。
入口：M1 文档与规范输出齐全。
退出：规范冲突闭环，关键约束清晰可落地。
交付物：规范评审记录、问题清单与修正建议。

M2 C++ 基础运行时
目标：基础模块正确、可测试、可复用。
覆盖：日志、配置加载、基于 Asio 的定时器封装、基于 Asio 的运行线程封装、基于 Asio 的事件循环、字节序/序列化、ZeroMQ 传输封装、统一消息封包、消息分发、心跳/注册编解码、错误码、metrics。
测试活动：功能测试、边界/异常输入测试、基础性能摸底（如 ZeroMQ 传输封装/序列化），并重点验证项目侧 Asio 封装的生命周期、取消、线程退出与停止语义，而不是重复验证上游库内部实现。
入口：M2 核心模块实现完成。
退出：关键功能稳定，异常处理可预期。
交付物：模块测试记录、问题与修复回归记录。

M3 进程与注册 Inner 网络
目标：GM/Gate/Game 生命周期与启动编排闭环可靠。
覆盖：注册表一致性、`NodeID` 唯一性、注册流程、心跳超时剔除、“所有节点已上线”通知、`Game -> Gate` 注册与心跳闭环、Game mesh ready 上报、`ServerStubEntity` ownership 下发、`Game` 服务就绪上报、`clusterReady` 状态下发、管理命令。
测试活动：开发期默认以 `1 GM + N Gate + M Game` 单机多进程方式验证启动/关闭/异常退出，并补充跨机部署下的注册心跳稳定性与管理命令正确性检查。
入口：GM/Gate/Game 主入口与注册逻辑完成。
退出：启动闭环稳定，异常情况下能正确恢复或降级。
交付物：集成测试记录、异常场景验证记录。

M4 网络与会话
目标：KCP 会话与基于已注册 `Game` 目录的 Gate↔Game 内部转发链路打通并可恢复。
覆盖：KCP 会话创建、会话表维护、基于启动期全连接建立的 `Game -> Gate` 注册目录稳定性、内部通信 / 请求转发与响应回传、推送通道、路由策略、负载感知分配、RPC 超时/重试、协议兼容性、心跳与重连。
测试活动：协议兼容性测试、并发会话测试、多 Gate × 多 Game 注册目录建立/断链恢复、断线/重连流程、网络波动场景验证。
入口：KCP 接入与转发链路实现完成。
退出：关键链路打通，异常网络条件下行为可预期。
交付物：网络与会话测试记录、兼容性测试记录。

M5 C# 分布式实体框架
目标：实体框架稳定，互操作可控。
覆盖：nethost/CLR 加载、C++↔C# 入口绑定、实体注册与分发、实体路由、Tick 驱动、持久化接口、Stub 示例、错误码映射、日志桥接。
测试活动：实体生命周期测试、路由与分发验证、互调边界场景验证。
入口：C# 框架与互操作接口实现完成。
退出：互操作稳定、实体路由正确、错误码与日志链路一致。
交付物：互操作验证记录、实体框架测试记录。

M6 闭环集成与质量
目标：端到端闭环、稳定性与性能基线达标。
覆盖：客户端→Gate→Game→C# 全链路、单机/多机部署下的配置与脚本可用性、日志/metrics 汇总、稳定性场景、性能基线、协议回归。
测试活动：端到端流程验证、单机与多机部署冒烟验证、稳定性长跑、性能基线测试、回归验证。
入口：闭环功能与脚本就绪。
退出：端到端通过、稳定性无阻断问题、性能基线满足目标。
交付物：端到端测试记录、稳定性与性能基线记录、回归记录。

**通用入口与退出标准**
入口：对应里程碑功能完成、接口稳定、可运行。
退出：P1/P2 缺陷关闭，关键场景通过，回归无新增阻断。

**主要风险与关注点**
1. 里程碑依赖一致性需要持续维护，尤其是文档、代码与第三方依赖基线之间的同步。
2. C++↔C# 互操作边界复杂，需要制定负面用例与回归清单。
3. ZeroMQ 内部链路与并发场景组合复杂，需要提前规划可重复的异常场景验证。
