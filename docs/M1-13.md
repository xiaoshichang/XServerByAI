# M1-13 设计说明（第三方依赖接入基线）

1. 当前项目采用 vendoring 方式接入基础第三方依赖，统一放在仓库根目录 `3rd/` 下，每个库独立一个子目录。
2. 当前确认的基础依赖为：`spdlog` 用于 native 日志模块，`zeromq/libzmq` 用于节点间基于 ZeroMQ over TCP 的内部通信。
3. 依赖源码应直接随仓库维护，当前基线版本为 `spdlog v1.17.0` 与 `libzmq v4.3.5`；构建阶段不依赖 `FetchContent` 或运行时联网下载。
4. 项目侧统一通过 `3rd/CMakeLists.txt` 接入依赖，并关闭上游示例、测试、文档与打包目标，避免把无关构建目标引入主工程。
5. 业务代码不直接依赖上游 target 命名；项目内部统一通过 `xs::thirdparty::spdlog` 与 `xs::thirdparty::zeromq` 这类别名引用依赖。
6. `3rd/README.md` 负责记录当前 pin 住的版本、上游来源与接入方式；升级依赖时必须同步更新该说明。
7. 后续涉及 native 日志、节点间传输、构建脚本、目录结构的文档与开发计划，应默认以本条目定义的第三方依赖基线为准。
