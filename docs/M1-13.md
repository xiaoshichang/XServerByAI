# M1-13 设计说明（第三方依赖接入基线）

1. 当前项目采用 vendoring 方式接入基础第三方依赖，统一放在仓库根目录 `3rd/` 下，每个库独立一个子目录。
2. 当前确认的基础依赖为：`spdlog` 用于 native 日志模块，`zeromq/libzmq` 用于节点间基于 ZeroMQ over TCP 的内部通信，standalone `asio` 用于 native 网络、事件循环、定时器与执行器基础设施，且不引入其他 Boost 组件。
3. 依赖源码应直接随仓库维护，当前基线版本为 `spdlog v1.17.0`、`libzmq v4.3.5` 与 `asio 1.36.0`；构建阶段不依赖 `FetchContent` 或运行时联网下载。
4. 项目侧统一通过 `3rd/CMakeLists.txt` 接入依赖；可编译的上游依赖应关闭无关示例、测试、文档与打包目标，header-only 依赖则通过项目侧 `INTERFACE` target 暴露头文件、编译宏与所需系统库。
5. 业务代码不直接依赖上游 target 命名或目录结构；项目内部统一通过 `xs::thirdparty::spdlog`、`xs::thirdparty::zeromq` 与 `xs::thirdparty::asio` 这类别名引用依赖。
6. `3rd/README.md` 负责记录当前 pin 住的版本、上游来源与接入方式；升级依赖时必须同步更新该说明。
7. 后续涉及 native 日志、网络/事件循环、定时器、线程运行封装、节点间传输、构建脚本、目录结构的文档与开发计划，应默认以本条目定义的第三方依赖基线为准。
8. `M2-03`、`M2-04`、`M2-05` 应以 standalone `asio` 的 `steady_timer`、`io_context`、executor / work guard 为实现基线；项目侧只补充进程生命周期、线程命名、取消与停止约束，不再自研独立定时器框架或事件循环内核。
