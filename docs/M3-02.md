# M3-02 设计说明（GM 生命周期与节点网络骨架）

1. `src/native/node` 移除 `NodeGmRunner`、`NodeGateRunner`、`NodeGameRunner`、`ServerNodeEnvironment`、`NodeRuntimeContext` 与 `NodeRuntime`。进程入口只保留 `node_main.cpp`，由 `NodeCreateHelper` 负责解析命令行并按 `selector` 构造具体节点对象。
2. 引入抽象基类 `ServerNode` 作为统一节点骨架。节点公共字段直接下沉到基类，包括 `config_path`、`selector`、`process_type`、`node_id`、`pid`、`NodeConfig`、`Logger`、`MainEventLoop` 与最近错误字符串；这些对象都在 `Init()` 内完成初始化，不再由额外环境对象注入。
3. `ServerNode` 对外只暴露 `Init()`、`Run()`、`Uninit()` 三个生命周期函数，统一返回 `NodeErrorCode`。详细错误文本通过 `last_error_message()` 查询，避免生命周期接口再混入输出参数或额外 runtime 包装层。
4. `ServerNode::Init()` 负责校验 `configPath` / `selector`、加载配置、记录 `pid`、初始化 logger 与主事件循环，并校验配置里的 `process_type` 是否符合当前节点角色。子类只需要在 `OnInit()` 中补充角色专属资源装配。
5. 角色实现拆成 `GmNode`、`GateNode`、`GameNode` 三个 `ServerNode` 子类。`GmNode` 当前负责 GM 控制面监听；`GateNode` 预留集群内通信与客户端通信双通道骨架；`GameNode` 预留集群内通信骨架。后续 `M3-08`、`M3-10`、`M4-02` 直接在对应子类内扩展，不再改写总入口。
6. 集群内部网络统一封装到 `InnerNetwork`。该类负责持有 `ZmqContext`、`ZmqPassiveListener`、监听状态、`configured_endpoint` / `bound_endpoint`、最近错误与 metrics，并暴露 `Init()`、`Run()`、`Uninit()` 生命周期。本阶段只交付被动监听模式，满足 GM 控制面监听并为后续 Gate/Game 复用打底。
7. 面向客户端的网络行为统一预留在 `ClientNetwork`。当前只提供最小占位生命周期实现，让 `GateNode` 的结构先稳定下来；真正的客户端监听、会话管理与 KCP 行为留给 `M4-02` 及后续条目继续填充。
8. 自动化验证覆盖两层：`node_create_helper_tests` 校验 CLI 解析、节点构造、`ServerNode` 生命周期与默认节点启动；`node_gm_node_tests` 校验 `GmNode` 对 GM 配置的约束，以及 `InnerNetwork` 的通配端口绑定、收包、状态收敛和日志输出。这样既保护新架构边界，也保留 M3-02 的行为回归。
