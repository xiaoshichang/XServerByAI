# M4-05A managed RPC 框架

## 当前状态
`M4-05A` 已经完成第一阶段的 entity RPC 骨架，目标是先把 managed 侧与 client 侧的声明式 RPC 能力打通，后续再在此基础上接入真实网络链路、属性同步与更高效的序列化方案。

## 已落地范围

### 1. 共享 RPC 契约
- 在 `src/managed/Foundation/Rpc/` 中新增共享 RPC 基础设施，供 managed framework 与 client framework 共用。
- 提供两组特性别名：
  - `[ServerRPC]` 与 `[ServerRpc]`
  - `[ClientRPC]` 与 `[ClientRpc]`
- 固定 entity RPC 内部消息号：
  - `ClientToServerEntityRpcMsgId = 6302`
  - `ServerToClientEntityRpcMsgId = 6303`
- 提供统一的 `EntityRpcInvocationEnvelope`、JSON codec、反射扫描缓存与错误码定义。

### 2. RPC 收集与参数绑定
- RPC 方法通过反射扫描收集，并按 entity 类型缓存元数据。
- 当前阶段默认以“方法名”作为 RPC 名称，不支持同名重载。
- 当前阶段只支持以下参数类型及其 nullable 版本：
  - `string`
  - `bool`
  - 各类整数
  - 浮点数
  - `Guid`
- 暂不支持复杂对象、`ref/out`、泛型方法、`params` 参数与返回值为非 `void` 的 RPC。

### 3. Client 侧接入方式
- `ClientEntity` 提供：
  - `CallServerRpc(string rpcName, params object?[] arguments)`
  - `CallServerRPC(string rpcName, params object?[] arguments)`
- `ClientEntity` 自身不直接处理网络发送，而是依赖运行时注入的 `IClientEntityRpcSender`。
- `ClientRuntimeState` 负责：
  - 在 entity 注册/解绑时维护 RPC sender 绑定
  - 在收到 `ServerToClientEntityRpcMsgId` 时，基于 `EntityManager` 查找目标 `ClientEntity`
  - 通过 `ClientEntityRpcDispatcher` 分发到 `[ClientRPC]` 方法
- `ClientEntity` 仍然只保存业务实体状态，不持有 `IsServerConfirmed`、`GameNodeId`、`SessionId` 这类底层网络/登录流程状态；这些状态继续留在 runtime/session 层。

### 4. Server 侧接入方式
- `ServerEntity` 提供：
  - `CallClientRpc(string rpcName, params object?[] arguments)`
  - `CallClientRPC(string rpcName, params object?[] arguments)`
- `ServerEntity.ReceiveProxyCall` 会拦截 `ClientToServerEntityRpcMsgId`，并通过 `ServerEntityRpcDispatcher` 反射分发到 `[ServerRPC]` 方法。
- `ServerEntity` 提供 `GetDefaultClientRpcTargetAddress()` 扩展点，具体实体可以决定默认回包目标地址。

### 5. AvatarEntity 首个样例
- server `AvatarEntity` 新增：
  - `[ServerRPC] void SetWeapon(string w)`
  - 设置当前装备后，通过 `CallClientRpc("OnSetWeaponResult", true)` 返回结果
- client `AvatarEntity` 新增：
  - `[ClientRPC] void OnSetWeaponResult(bool succ)`
  - 用于记录最近一次设置武器是否成功
- 当前使用方式与目标示例一致：

```csharp
// client
public void OnBtnClick()
{
    self.CallServerRpc("SetWeapon", "gun");
}

[ClientRPC]
public void OnSetWeaponResult(bool succ)
{
    if (succ)
    {
        // log(...)
    }
}

// server
[ServerRPC]
public void SetWeapon(string w)
{
    this.CallClientRpc("OnSetWeaponResult", true);
}
```

## 错误模型与日志
- 当前阶段定义了统一的 `EntityRpcDispatchErrorCode`，覆盖：
  - payload 非法
  - 目标实体不存在
  - RPC 不存在
  - 参数个数不匹配
  - 参数类型不支持
  - 参数绑定失败
  - 目标方法执行失败
- server/client 两侧分发失败时都会产出统一格式的错误消息，便于后续接入真实链路时继续复用。

## 暂不包含的内容
- `M4-06` 负责把 client 发起的 entity RPC 真正接入 `Client -> Gate -> Game` 链路。
- `M4-07` 负责把 server 返回的 entity RPC 真正接入 `Game -> Gate -> Client` 链路。
- 当前 JSON 只是第一阶段实现，后续如需切换为二进制或其他高效格式，应只替换 codec 层，不改变 entity 侧 API。
- 属性同步、批量推送、超时重试等增强能力不在本条目范围内。
