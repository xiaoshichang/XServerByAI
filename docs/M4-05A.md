# M4-05A Managed RPC 框架

## 目标
`M4-05A` 负责先把 entity RPC 的框架搭起来，让 client 与 server 都可以用统一的声明方式定义 RPC；网络接入、真实链路打通和更复杂的同步能力则由后续条目继续扩展。

## 当前实现

### 1. 共享 RPC 代码位置
- 共享源码物理位置为 `src/shared/EntityRpc/`。
- `client/XServer.Client.Framework` 与 `src/managed/Framework` 都通过链接源文件的方式，把这些共享文件映射到各自项目里的 `Rpc/Shared/` 视图目录。
- RPC 相关逻辑属于 framework，而不是 foundation。

### 2. 宏与命名
- client 侧编译宏：`XSERVER_CLIENT_FRAMEWORK`
- server 侧编译宏：`XSERVER_SERVER_FRAMEWORK`
- RPC 反射扫描器名称：`EntityRpcMethodScanner`

### 3. 固定消息号
- `ClientToServerEntityRpcMsgId = 6302`
- `ServerToClientEntityRpcMsgId = 6303`

### 4. 收集与绑定方式
- RPC 方法通过反射扫描收集。
- 参数类型与签名也通过反射确定。
- 当前序列化格式先使用 JSON，后续如需替换为更高效格式，只需要调整 codec 层。
- 当前阶段不支持同名重载 RPC。

### 5. Client 侧接口
- `ClientEntity.CallServerRpc(string rpcName, params object?[] arguments)`
- `ClientEntity.CallServerRPC(string rpcName, params object?[] arguments)`
- `ClientRuntimeState.ConfigureRpcSender(...)` 负责为当前已注册实体注入 sender。
- `ClientEntity` 自身只保存业务状态，不保存 `IsServerConfirmed`、`GameNodeId`、`SessionId` 这类网络 / 登录流程状态。

### 6. Server 侧接口
- `ServerEntity.CallClientRpc(string rpcName, params object?[] arguments)`
- `ServerEntity.CallClientRPC(string rpcName, params object?[] arguments)`
- `ServerEntity.ReceiveProxyCall(...)` 会在收到 `6302` 时自动分发到 `[ServerRPC]` 方法。

## AvatarEntity 示例

### Client
```csharp
public void OnBtnClick()
{
    self.CallServerRpc("SetWeapon", "gun");
}

[ClientRPC]
public void OnSetWeaponResult(string weapon, bool succ)
{
    if (succ)
    {
        // log(...)
    }
}
```

### Server
```csharp
[ServerRPC]
public void SetWeapon(string weapon)
{
    this.CallClientRpc("OnSetWeaponResult", weapon, true);
}
```

## 当前边界
1. 该条目只定义 RPC 框架本身，不负责完整的端到端网络链路。
2. 属性同步、批量推送、重试、超时管理等增强能力不在本条目范围内。
3. 真正的 `Client -> Gate -> Game` 链路由 `M4-06` 负责接通。
