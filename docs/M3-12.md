# M3-12 GM 下发 allNodesOnline 通知设计

本文档定义 `M3-12` 的实现范围：`GM` 在期望 `Game` / `Gate` 节点完成到 `GM` 的注册后，向全部 `Game` 下发 `Inner.ClusterNodesOnlineNotify (1204)`，并在结论变化时同步当前最新状态。

## 目标与边界

1. 本特性只解决 `GM -> Game` 的“所有节点已上线”聚合与通知，不在本里程碑内实现 `Game -> Gate` 全连接闭环、mesh ready 上报、ownership 下发或 `clusterReady` 聚合。
2. 当前阶段不引入额外轮次字段；`allNodesOnline` 只表达最新聚合结论，不额外携带轮次容错语义。
3. `GM` 对“在线”的判断以当前注册表和心跳超时状态为准；`Game` 与 `Gate` 不允许自行推断“所有节点已上线”。
4. `Game` 当前阶段只消费并记录 `allNodesOnline` 最新结论，把它作为后续 `M3-13` 启动 `Game -> Gate` 闭环的唯一触发信号。

## 实现清单

1. 补齐 `1204` 通知编解码
   - 在现有启动编排消息编解码基础上补充 `Inner.ClusterNodesOnlineNotify` 的 `msgId`、消息体结构、线长常量以及 encode/decode 校验。
   - 消息体字段遵循简化后的协议语义：`allNodesOnline`、`statusFlags`、`serverNowUnixMs`；其中 `statusFlags` 当前必须为 `0`。
   - 数据包封套使用单向通知语义：`flags = 0`、`seq = kPacketSeqNone`，与 `Gate` 侧消费 `ClusterReadyNotify` 的做法保持一致。

2. 建立 `GM` 的期望节点集合与在线聚合
   - `GM` 在初始化阶段从集群配置中提取期望 `Game` / `Gate` 节点列表，后续聚合只基于这组节点，不把 `GM` 自己计入在线判定。
   - 只有当期望节点都存在注册会话，且会话处于已注册、未心跳超时的有效状态时，聚合结论才为 `allNodesOnline = true`。
   - 聚合逻辑在注册成功、心跳超时、旧会话被替换等会改变在线性的路径后统一重算，避免不同代码路径各自维护结论。

3. 维护 `GM` 的广播策略
   - `GM` 新增启动编排状态，至少持有当前 `allNodesOnline` 结论和最近一次通知的服务器时间。
   - 当聚合结论发生变化时，`GM` 向当前所有已注册 `Game` 广播最新通知，确保它们都以同一份最新结论启动或撤销后续闭环。
   - 当聚合结论未变化时，`GM` 不重复下发通知，只记录告警日志，避免把注册重试误当成新的编排事件。

4. 补齐 `Game` 对 `allNodesOnline` 的接收与状态落地
   - `GameNode` 增加 `allNodesOnline` 与最近一次 `serverNowUnixMs` 的本地状态，默认值为未收到通知。
   - `HandleGmMessage` 需要像 `GateNode` 对 `ClusterReadyNotify` 那样，先识别 `1204` 单向通知，再进行普通响应包的 `seq` 校验，避免把 `seq = kPacketSeqNone` 的通知误判为非法响应。
   - `Game` 仅接受在 GM 注册完成后到达的通知；成功消费后覆盖本地状态并输出日志，为 `M3-13` 接入被动 `Gate` 闭环做好挂点。

5. 回归测试与文档同步
   - 为 `1204` 通知增加独立编解码测试，覆盖正常编码解码、`statusFlags != 0`、布尔值非法、缓冲区不足和尾随字节等场景。
   - 为 `GM` 增加节点编排测试，覆盖“未齐节点不发送 true”、“全部期望节点上线后广播 true”、“心跳超时后广播 false”以及“结论未变化时不重复广播”。
   - 为 `Game` 增加 GM 会话测试，覆盖合法通知接收、非法封套拒收、注册前通知拒收以及重复通知幂等处理。
   - 同步更新长期文档里的轮次说明，统一为当前阶段的简化编排语义。

## 验证出口

1. `GM` 在只有部分期望节点注册成功时，不得向 `Game` 下发 `allNodesOnline = true`。
2. 期望 `Gate` / `Game` 节点全部完成注册后，所有在线 `Game` 都能收到 `allNodesOnline = true`。
3. 任一期望节点因心跳超时失效后，`GM` 会向已注册 `Game` 下发 `allNodesOnline = false`。
4. `Game` 忽略非法封套的通知，且 `GM` 在聚合结论未变化时不会重复广播。
