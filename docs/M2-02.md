# M2-02 设计说明（配置加载）

1. 在 `src/native/core` 内落地独立配置模型与加载入口，对外暴露不依赖 `nlohmann/json` 细节的结构体和函数，至少覆盖 `env`、`logging`、`kcp`、`gm`、`gate`、`game` 这组当前规范要求的逻辑块。
2. 配置加载固定读取单个 UTF-8 JSON 文件，先完成整文件语法解析与根对象校验，再做结构化映射；任何根对象、顶层必填块或所选实例缺失都应返回稳定错误，而不是静默退回默认值。
3. `nodeId` 解析统一收敛到 core 公共函数，支持 `GM`、`Gate<index>`、`Game<index>`，并在加载阶段完成“`nodeId` -> 实例块 -> `NodeID`”的一致性校验，例如 `gate.Gate0` 必须对应 `Gate0`。
4. `logging` 解析固定为顶层公共块；当前阶段不支持实例级 `logging` 覆盖，合并结果继续复用 `M2-01` 已落地的 `LoggingConfig` 与 `ValidateLoggingConfig`。
5. 顶层 `kcp` 与 `game.*.managed` 按当前文档基线映射为独立结构；`managed.assemblyName` 缺失时回落到当前固定根程序集名 `XServer.Managed.GameLogic`。Gate 实例块必须同时提供 `innerNetwork` 与 `clientNetwork`。
6. 未知顶层块与未知 `logging` 字段继续按规范直接失败；其余已建模对象也采用显式字段校验，避免拼写错误被静默吞掉，同时把错误消息尽量定位到具体 JSON 路径。
7. `xserver-node` 在参数校验通过后立即调用配置加载逻辑，当前阶段只要能完成文件解析、实例选择和结构校验即可视为启动前检查通过，进程主体生命周期仍留给后续条目实现。
8. 自动化验证复用现有 native 最小测试入口，至少覆盖：成功加载 `gm/gate/game`、Gate `clientNetwork` 必填、未知顶层块失败、未知字段失败，以及非法 `NodeID` 失败。
