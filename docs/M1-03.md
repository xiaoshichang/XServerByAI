# M1-03 设计说明（进程入口可执行占位）

1. 在 `src/native/node` 下提供统一的节点入口目标，最终生成单个可执行文件 `xserver-node`。
2. 统一入口的最小命令行形式固定为 `xserver-node <configPath> <selector>`，其中 `selector` 当前支持 `gm`、`gate0`、`game0` 这类实例选择器。
3. Gate / Game 的 `selector` 只负责定位启动实例；稳定运行时身份仍复用配置里的 `nodeId` 语义，例如 `gate0` 对应 `Gate0`、`game1` 对应 `Game1`。
4. `xserver-node` 入口目标继续链接 `core`、`net`、`ipc`、`host` 以及节点静态库，确保后续生命周期、配置加载与日志初始化逻辑都能从统一入口展开。
5. GM、Gate、Game 的节点实现统一收敛在 `src/native/node/` 内部，由共享的 `node` 静态库编译，不再拆成 `src/native/gm`、`src/native/gate`、`src/native/game` 独立库目标。
