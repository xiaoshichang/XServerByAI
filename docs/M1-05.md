# M1-05 设计说明（目录结构、命名与命名空间规则）

1. 将目录结构、命名与命名空间规则沉淀为长期维护的规范文档，而不是只写在里程碑说明里。
2. 约定仓库顶层目录职责，补充 `3rd/` 作为 vendored 第三方依赖的固定落点，避免后续新增目录时出现语义重叠。
3. 约定 `src/native` 与 `src/managed` 的目录命名方式、目标命名方式和文件放置规则。
4. 明确第三方代码不得直接混入业务源码目录；每个依赖都应在 `3rd/<libname>/` 下独立维护源码与必要的项目侧构建包装。
5. 基于现有实现明确 C++ 使用 `xs` 根命名空间，C# 使用 `XServer.Managed` 根命名空间。
6. 补充文档命名规则与新增模块时需要同步更新的文档入口，降低后续扩展时的分歧成本。
7. 在 `PROJECT.md` 中保留简要目录概览，并将细则统一指向独立规范文档。
