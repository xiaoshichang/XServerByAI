# M1-06 设计说明（二进制协议包头结构与常量）

1. 将包头结构与常量沉淀为 `docs/` 下长期维护的协议文档，避免后续序列化实现各自解释。
2. 明确包头字段长度、网络字节序、固定头长与魔数/版本/标志位语义。
3. 在 `src/native/net` 中提供可被后续编解码逻辑直接复用的 C++ 结构与常量定义。
4. 在 `src/managed/Foundation` 中提供语义一致的 C# 结构与常量定义，供后续互操作与业务侧共享。
5. 只定义当前确定的头部协议，不提前引入 `crc`、`traceId` 等尚未落地的扩展字段。
6. 通过 native 与 managed 的最小构建验证，确认协议定义本身可以被后续里程碑直接消费。
