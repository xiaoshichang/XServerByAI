# M1-15 设计说明（C++↔C# 互操作入口签名与约定）

1. 将当前阶段 native host 与 managed `GameLogic` 之间的互操作入口签名、装载顺序与 ABI 约定沉淀为 `docs/MANAGED_INTEROP.md` 的长期维护文档。
2. 明确当前阶段仅 `Game` 进程宿主 CLR，根程序集固定为 `XServer.Managed.GameLogic`，入口导出类型固定为 `XServer.Managed.GameLogic.Interop.GameNativeExports`。
3. 固定首版导出入口集合：`GameNativeGetAbiVersion`、`GameNativeInit`、`GameNativeOnMessage`、`GameNativeOnTick`，并要求 native 在首次业务调用前完成 ABI 版本校验。
4. 统一调用约定为 `cdecl` + `UnmanagedCallersOnly`，约束导出方法必须为静态、不可重载、不可跨边界抛异常。
5. 定义 `ManagedInitArgs` 与 `ManagedMessageView` 两个最小 blittable 结构，统一字段类型、字符串编码、缓冲区借用语义与对齐原则，为后续 `M6-01` 封送实现提供稳定边界。
6. 明确调用时序与线程语义：`Init` 成功前不得调用 `OnMessage` / `OnTick`，当前默认由 native 保证非重入串行调用，managed 侧不得缓存 native 输入指针。
7. 在 `docs/PROJECT.md` 与 `docs/ERROR_CODE.md` 中同步补充引用，确保总览描述、错误码预留与互操作 ABI 文档保持一致。
